The sandwich package implements the Sandwich model-based mappling1 based on the spatial stratified heterogeneity (SSH) theory2. It consists of sampling, SSH, and reporting layers. The goal of this model is to estimate the mean value of the sampling attribute and its standard error for each reporting unit leveraging the known distribution of the samples in the SSH layer. Since the Sandwich model does not rely on the spatial dependence of the population, when the spatial dependence is weak, Sandwich is still applicable as long as one properly stratifies the geospatial surface. Figure 1 is an overview of the Sandwich model structure.

Figure 1. Structure of the Sandwich model.

Loading package

To use the sandwich package, first install it using install.packages(), and then load the installed package into your environment using library().

# install.packages("sandwich")
library("sandwich")

Loading data

The Sandwich model typically takes three inputs: sampling, SSH, and reporting layers. To successfully run the model, all the input data should be sf (simple feature; see Simple Features for R for references) objects. As an example, here, we import a built-in dataset collected in Heshun county, China. The load.shp function can help prepare a shapefile into a sf object.

hs.sampling = load.shp("./data", "hs.sampling.shapefile")
hs.ssh = load.shp("./data", "hs.ssh.shapefile")
# here's another candidate SSH layer
hs.ssh2 = load.shp("./data", "hs.ssh2.shapefile")
hs.reporting = load.shp("./data", "hs.reporting.shapefile")

We use ggplot2 to visualize this dataset.

# sampling layer
ggplot(data=hs.sampling) + geom_sf(aes(color=Population)) + 
  ggtitle("Sampling Layer") + theme(plot.title=element_text(hjust=0.5))

# SSH layer
ggplot(data=hs.ssh) + geom_sf(aes(fill=STR_1)) + labs(fill="Code") + 
  ggtitle("SSH Layer A") + theme(plot.title=element_text(hjust=0.5))

# another SSH layer
ggplot(data=hs.ssh2) + geom_sf(aes(fill=STR_2)) + labs(fill="Code") + 
  ggtitle("SSH Layer B") + theme(plot.title=element_text(hjust=0.5))

# reporting layer
ggplot(data=hs.reporting) + geom_sf(aes(fill=CODE)) + labs(fill="Code") +
  ggtitle("Reporting Layer") + theme(plot.title=element_text(hjust=0.5))

Selection of the SSH layer(s)

The accuracy of the Sandwich model is determined by the SSH layer. For an ideal SSH layer, values of the target attribute are expected to be homogeneous within each stratum and differ between the strata. To determine a proper SSH layer for the Sandwich model, the geographical detector model3 is applied to quantify the SSH of the target attribute with regard to the candidate stratification(s).

First, we combine the sampling and the candidate SSH layers to a single data frame.

hs.join = geodetector.data(hs.sampling, hs.ssh, "STR_1")
hs.join = geodetector.data(hs.join, hs.ssh2, "STR_2")

The factor detector q-statistic in the geographical detector model indicates the SSH of the sampling attribute in terms of a given stratification. The range of the q-statistic is between 0 and 1; a higher value of q denotes a higher level of SSH. Comparing the q-statistics derived under different stratifications can help selecting a proper SSH layer for Sandwich model-based mapping.

geodetector.factor(hs.join, "Population", c("STR_1", "STR_2"))
## [[1]]
##       q-statistic      p-value
## STR_1   0.9115915 4.320144e-10
## 
## [[2]]
##       q-statistic      p-value
## STR_2   0.3941722 1.515072e-10

The output of geodetector.factor above implies that hs.ssh (q=.912) has a higher determinant power over the attribute compared to hs.ssh2 (q=.394). Therefore, it will be more reasonable to select hs.ssh as the SSH layer for Sandwich model-based mapping.

The interaction detector in the geographical detector model quantifies whether a combination of two stratifications enhances the SSH of the sampling attribute. The q-statistic derived under a combination of different stratifications indicates whether the determinant power is significantly raised by the combined effect of two candidate SSH layers. This provides useful information of the potential optimization of existing candidate SSH layers.

geodetector.interaction(hs.join, "Population", c("STR_1", "STR_2"))
##      [,1]    [,2]    [,3]               
## [1,] "STR_1" "STR_2" "0.920244824766666"
## [2,] "STR_1" "STR_1" "0.911591467802692"
## [3,] "STR_2" "STR_2" "0.394172207833932"

Since the combined q-statistic of hs.ssh and hs.ssh2 (q=.920) is approximately the same as that of using hs.ssh alone (q=.912), we can optimize the SSH layer of the Sandwich model by combining hs.ssh and hs.ssh2 or simply use hs.ssh as the SSH layer. For the purpose of simplicity, we will use hs.ssh alone as the SSH layer in the subsequent modeling.

Running the Sandwich model

The Sandwich model-based mapping is performed using the sandwich.model function, which outputs the mean value of the sampling attribute and its standard error for each reporting unit.

hs.sw = sandwich.model(hs.sampling, hs.ssh, hs.reporting, "Population")

Model validation

To evaluate the overall accuracy of the Sandwich model, a k-fold cross validation can be performed using the sandwich.cv function. A diagnostic statistic called root mean square error (RMSE) will be calculated. Here, we present the result of a 5-fold cross validation.

hs.cv = sandwich.cv(hs.sampling, hs.ssh, hs.reporting, "Population", k=5)
hs.cv
## [1] 100.5524
